# Interpretando a intenção do usuário

Antes de agir, classifique o pedido:

- **Dúvida / conhecimento**: explique de forma direta e objetiva
- **Nova funcionalidade ou refatoração**: siga o fluxo de tarefas (task.md)
- **Correção de bug**: siga o fluxo de bugs (bug.md)

## Regras gerais de comportamento

- Entenda o contexto completo antes de escrever código
- Se a tarefa for vaga, identifique os gaps e mencione antes de implementar
- Funcionar primeiro, refatorar depois
- Simples agora, sustentável amanhã
- Proibido comentar código óbvio
- Proibido `console.log` em código de produção
